# 03 - تصميم قسم العملاء والطلبات

## الهدف

تحويل صفحة العملاء والطلبات إلى مركز إدارة عملاء عملي، وليس مجرد كروت.

## الهيكل المقترح

### 1. Header عملي

يحتوي:

- عنوان: العملاء والطلبات
- وصف قصير: إدارة العملاء، الطلبات، والتفضيلات
- زر أساسي: إضافة عميل
- زر ثانوي عند الحاجة: إضافة طلب

### 2. شريط بحث وفلاتر

العناصر:

- بحث بالاسم أو المصدر أو نوع المحتوى.
- فلتر الحالة:
  - الكل
  - نشط
  - يحتاج متابعة
  - مكتمل
  - مؤرشف
- فلتر المصدر:
  - YouTube
  - Instagram
  - WhatsApp
  - Discord
  - Other

## شكل قائمة العملاء

الأفضل استخدام قائمة/جدول مرن بدل كروت ضخمة فقط.

كل عميل يعرض:

- صورة أو initials.
- الاسم.
- نوع المحتوى.
- المصدر.
- آخر نشاط.
- عدد الطلبات النشطة.
- أهم تفضيل مختصر.
- حالة العميل.
- زر فتح مساحة العمل.

## حالات العميل المقترحة

- جديد
- نشط
- ينتظر رد
- عميل دائم
- VIP
- مؤرشف

## مثال كرت عميل محسّن

محتوى الكرت:

- الاسم: ينال الخياط
- النوع: Gaming / Commentary
- المصدر: YouTube
- آخر طلب: Thumbnail لفيديو جديد
- الحالة: ينتظر مراجعة
- تفضيلات: نص كبير، ألوان قوية، وجوه واضحة
- الإجراء: فتح مساحة العمل

## أزرار مسموحة في هذه الصفحة

- إضافة عميل
- فتح مساحة العميل
- إضافة طلب للعميل
- أرشفة العميل
- تعديل بيانات مختصرة

## أزرار يجب حذفها

- أي زر دفع.
- أي زر فواتير.
- أي زر تحليلات مالية.
- أي زر لا يغير بيانات أو يفتح شاشة مفيدة.

## Empty State

عندما لا يوجد عملاء:

- رسالة واضحة: لا يوجد عملاء بعد.
- زر: إضافة أول عميل.
- لا حاجة لكروت أو رسومات كثيرة.

## Mobile Behavior

على الموبايل:

- تتحول القائمة إلى كروت compact.
- البحث يبقى أعلى الصفحة.
- زر إضافة عميل ثابت أو واضح.
- لا تظهر كل الأعمدة، فقط الأهم:
  - الاسم
  - الحالة
  - آخر طلب
  - فتح
