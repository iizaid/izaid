# 04 - مساحة عمل العميل

## الهدف

مساحة عمل العميل يجب أن تكون أهم أداة في النظام. وظيفتها أن تجمع كل ما تحتاجه قبل التصميم وأثناء التسليم.

## الهيكل العام

تقسم الصفحة إلى Tabs أو أقسام واضحة:

1. Overview
2. Client Profile
3. Style Preferences
4. Requests
5. Deliverables
6. Notes / Timeline

## 1. Overview

يعرض ملخص سريع:

- اسم العميل.
- صورة أو لوقو.
- نوع المحتوى.
- روابط التواصل.
- الطلب الحالي.
- حالة الطلب الحالي.
- آخر ملاحظة مهمة.
- Quick actions:
  - إضافة طلب
  - رفع تسليم
  - نسخ رابط المعاينة

## 2. Client Profile

معلومات ثابتة عن العميل:

- الاسم.
- البريد إن وجد.
- رقم واتساب أو وسيلة تواصل.
- رابط قناة YouTube.
- رابط Instagram.
- رابط Discord إن وجد.
- نوع المحتوى.
- الجمهور المستهدف.
- اللغة المستخدمة.
- ملاحظات شخصية أو عملية.

## 3. Style Preferences

هذا القسم يجب أن يكون أقوى بكثير من الوضع الحالي.

### الحقول المقترحة

- الألوان المفضلة.
- الألوان الممنوعة.
- الخطوط المفضلة.
- نمط التصميم:
  - Gaming
  - Documentary
  - Tech
  - Clean
  - Dramatic
  - Business
- حجم النص:
  - قليل
  - متوسط
  - كبير
- كثافة التصميم:
  - minimal
  - balanced
  - intense
- استخدام الوجوه:
  - نعم
  - لا
  - حسب الفيديو
- الكلمات أو الأساليب التي لا يحبها العميل.
- روابط أمثلة مرجعية.

## 4. Requests

كل طلب يجب أن يظهر كعنصر واضح.

### بيانات الطلب

- عنوان الطلب.
- brief.
- الحالة.
- موعد التسليم.
- عدد النسخ المطلوبة.
- ملاحظات التصميم.
- روابط الفيديو أو المصادر.

### حالات الطلب

- Brief
- Designing
- Waiting Review
- Revision
- Delivered
- Approved

## 5. Deliverables

قسم التسليم الاحترافي.

### المطلوب

- رفع حتى 3 نسخ أو أكثر حسب الطلب.
- عنوان لكل نسخة.
- ملاحظات لكل نسخة.
- حالة كل نسخة:
  - Draft
  - Final
  - Approved
- زر تحميل.
- زر نسخ رابط المعاينة.
- اختيار النسخة المعتمدة.

## 6. Notes / Timeline

سجل بسيط لكل ما حدث:

- تم إنشاء العميل.
- تم إضافة طلب.
- تم تعديل التفضيلات.
- تم رفع نسخة.
- تم اعتماد نسخة.
- ملاحظة داخلية.

## أشياء يجب حذفها من مساحة العميل

- مكتمل الدفع.
- فواتير.
- إحصائيات مالية.
- روابط تسليم وهمية لا تعمل.
- upload غير محفوظ إذا كان يوحي أنه تسليم حقيقي.

## التصميم المقترح

- أعلى الصفحة: معلومات العميل + actions.
- يمين/يسار حسب RTL: tabs واضحة.
- وسط الصفحة: محتوى التبويب.
- استخدام cards فقط للأشياء المتكررة مثل الطلبات والنسخ.
- لا يوجد cards داخل cards.
- ألوان هادئة مع accent أخضر/تركوازي.

## مبدأ مهم

إذا كانت وظيفة غير جاهزة backend، لا تظهر كأنها جاهزة في الواجهة.

مثال:

- إذا التسليم لا يحفظ الصور، يظهر كـ local preview فقط أو يؤجل حتى Phase 3.
- إذا رابط المعاينة لا يعمل، لا يظهر زر نسخ رابط المعاينة.
